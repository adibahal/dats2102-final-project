## About this project

This project looks at the relationship between primary care provider access and health outcomes across US counties, using the 2025 County Health Rankings release. The central question is whether primary care provider access predicts health outcomes at the county level, and where counties defy the trend.

The analysis covers 2,795 US counties after cleaning. It builds from the distribution of provider access through stratified regressions and ends with a county-level residual map showing where premature death departs from what provider access alone would predict. Three regional clusters of underperformance show up clearly: tribal lands in the northern Plains and Southwest, Appalachian Kentucky and West Virginia, and the Mississippi Delta.

This project is submitted primarily under Track B (Map-Based Analysis), with supporting elements from Track C.

## Data

The primary dataset is the 2025 County Health Rankings & Roadmaps release, published by the University of Wisconsin Population Health Institute and the Robert Wood Johnson Foundation. Available at https://www.countyhealthrankings.org/health-data/methodology-and-sources/data-documentation. Distributed under the CC BY-SA 4.0 license. Accessed April 2026.

The secondary dataset is the 2023 USDA Rural-Urban Continuum Codes from the USDA Economic Research Service, available at https://www.ers.usda.gov/data-products/rural-urban-continuum-codes. Public domain. Accessed April 2026. This file was downloaded during early planning but ultimately not used in the final analysis; the % Rural variable from CHR was used instead.

Both files are included in the submission folder, in the same directory as the notebook.

## Reproducing the analysis

The notebook was written for Python 3.11. The packages used are pandas, numpy, matplotlib, seaborn, plotly, folium, and requests, plus openpyxl for reading the .xlsx files. All standard. To reproduce, keep the two data files in the same folder as the notebook and run the cells top to bottom. The notebook runs cleanly without errors. No random operations are involved, so no seed setting is needed.

The Plotly and Folium maps fetch a county GeoJSON file from a public Plotly-hosted URL the first time they run, so an internet connection is required for that one fetch.

A note on projections, since Track B asks for it: the static and interactive choropleths use Plotly's bundled Albers USA composite projection, which handles the Alaska and Hawaii inset placement automatically. The Folium layered map uses the default Web Mercator. No custom CRS transformations were performed because the source data was keyed on FIPS codes rather than raw geometries.
